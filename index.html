hexo博客搭建

为什么要搭建hexo博客

hexo博客是传播较为广泛的一个博客平台，一些常见的问题能快速找到解决办法抗因此稳定性很强

搭建hexo博客

  • 安装node.js插件（地址https://nodejs.org/zh-cn/）
  • 检查node版本
    • 打开终端，使用sudo命令将用户身份切换为管理员
    • 输入“node -v”查看版本
  • 切换为淘宝源，终端输入“npm install -g cnpm –registry=https://registry.npm.taobao.org”
  • 利用cnpm全局安装hexo博客系统，终端输入“cnpm install -g hexo-cli”
  • 利用“hexo -v”命令确定博客系统是否安装

至此博客平台配置完毕

建立hexo博客系统

hexo博客在本地要有载体，因此需要在本地创建博客存放目录，后续hexo博客的所有操作均在此目录下进行

  • 以管理员权限建立目录并利用cd命令进入到目录中
  • 输入“sudo hexo init”创建hexo博客项目
    • 此处出现了未安装git的错误提示，解决办法是启动xcode软件，用xcode下载git组件具体步骤如下
    • 直接从AppStore安装Xcode，Xcode集成了Git，不过默认没有安装，你需要运行Xcode，选择菜单“Xcode”->“Preferences”，在弹出窗口中找到“Downloads”，选择“Command Line Tools”，点“Install”就可以完成安装了。
  • 输入“hexo s”以启动hexo博客，命令中的s代表了Start
  • 输入“ctrl+c”结束进程

说明：“hexo s”命令为预览博客系统，用以查看最终效果

建立文章

  • “hexo n”中的n代指new意思是新建，整句格式如下“hexo n “标题””需要注意的是标题引号为英文
  • 将新建的文章编制完成后对整个项目进行编译
    • 清理：hexo clean
    • 编译：hexo g

推送至远端

博客系统部署成功之后需要将其部署至远端运行，由于此系统是静态博客所以只要将渲染好的网页推送出去就行了，通常所采用的远端仓库为github和gitee

  • 安装插件：cnpm install –save hexo-deployer-git
  • 更改配置文件：vim _config.yml
    • deploy:
    • type: ‘git’
    • repo: 仓库地址
    • branch: master
  • 部署：hexo d
    • 在部署过程中会弹出登陆提示，此时输入远端仓库的账号和密码即可完成推送

美化

可选用模板进行配置，此处推荐yilia

  • 命令：git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia
